## Basic Setup

Configuring your VB Mgzn documentation space involves setting up projects, teams, and access controls. Start with the fundamental settings to get your workspace operational.

<Steps>
  <Step title="Create Project" icon="folder">
    Set up a new documentation project with a unique name and description.
  </Step>
  <Step title="Configure Team" icon="users">
    Add team members and assign roles like admin, editor, or viewer.
  </Step>
  <Step title="Set Permissions" icon="shield">
    Define access levels for different content areas and API endpoints.
  </Step>
</Steps>

## Environment Variables

Use environment variables to configure sensitive settings and deployment-specific options.

<CodeGroup tabs="bash,powershell">
  ```bash
export VB_MGZN_API_KEY="your_api_key"
export VB_MGZN_BASE_URL="https://api.vbmgzn.com"
export VB_MGZN_PROJECT_ID="your_project_id"
```
  ```powershell
$env:VB_MGZN_API_KEY = "your_api_key"
$env:VB_MGZN_BASE_URL = "https://api.vbmgzn.com"
$env:VB_MGZN_PROJECT_ID = "your_project_id"
```
</CodeGroup>

<Callout kind="alert">Never commit sensitive configuration values to version control. Use environment variables or secure vaults instead.</Callout>
